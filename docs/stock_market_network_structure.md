# 股票市场网络结构分析系统 · 说明书

> 用图论 / 网络分析方法，从相关性结构中发现**抱团簇、核心龙头、趋势相变**。
> 本文档作为后续编码的需求规格说明（spec），覆盖：背景 → 分析过程 → 结论 → 执行计划。

---

## 1. 背景

### 1.1 问题动机

相关性矩阵本身有 N×N 个数字，信息过载且充满噪声，人无法直接解读。
网络分析的本质是**从相关性矩阵中提取出最有信息量的"骨架"**，把高维相关结构压缩成可解读的拓扑，从而回答三个实际问题：

1. **抱团**：市场里哪些股票在异常集中地联动（资金虹吸）？
2. **核心龙头**：谁是结构中心？谁是趋势引领者（先动、别人跟）？
3. **趋势**：抱团/市场结构如何随时间演化，何时出现相变（形成、致密、瓦解）？

### 1.2 方法谱系（econophysics 成熟方向，20+ 年积累）

- Mantegna (1999)：奠基，提出基于相关性的最小生成树（MST）
- Onnela et al. (2003)：动态资产树，研究网络随时间演化
- Tumminello et al. (2005)：PMFG（平面最大过滤图）
- López de Prado (2016)：HRP（层次风险平价），网络/聚类思想在组合管理中最成功的落地

### 1.3 核心思想

- **节点** = 股票
- **边** = 股票间关系（最常用：收益率相关性）
- 整个市场 → 一张图 → 图论全套工具（中心性、社区检测、最小生成树、拓扑指标）皆可用

## 2. 分析过程

### 2.1 建网络（最关键的一步）

**Step 1 — 相关性矩阵**
- 输入：复权收盘价（adjusted close，必须处理分红/拆股）
- 计算对数收益率 → 全部股票对的 Pearson 相关系数 ρᵢⱼ
- 估计稳健性：考虑 Ledoit-Wolf 收缩估计，或 RMT 去噪后再用

**Step 2 — 相关性转距离**（Mantegna 距离，满足度量公理）

```
d_ij = sqrt(2 * (1 - ρ_ij))
```

- ρ=1 → d=0（最近）；ρ=-1 → d=2（最远）

**Step 3 — 过滤 / filtering（精髓，去噪保骨架）**

| 方法 | 保留边数 | 特点 |
|------|---------|------|
| **MST**（最小生成树） | N-1 | 最经典、极简、清晰看层级与中心；但过稀疏，丢失环/社区信息 |
| **PMFG** | 3(N-2) | 保留更多社区与环结构，信息更丰富 |
| **阈值法** | 取决于阈值 | 简单但阈值主观 |
| **RMT 去噪 + 网络** | — | 用随机矩阵理论剔除符合随机预期的特征值（噪声），最严谨 |

> 维度灾难提醒：当股票数 N 接近样本长度 T 时，相关矩阵估计极不可靠，此时 RMT 去噪几乎是**必须**的。经验：T/N ≥ 2，最好 ≥ 5。

### 2.2 分析工具

**(A) 节点中心性 —— 找"结构核心"**
- 度中心性 / 加权度（degree / strength）
- 介数中心性（betweenness）：跨板块枢纽，风险传染关键
- 特征向量中心性 / PageRank：系统重要性
- ⚠️ 相关性是**对称**的，这种中心性衡量的是"最有代表性、和大家最同步"的票——往往是市值最大、最像板块 ETF 的票，是"结构枢纽"≠"引领者"

**(B) 社区检测 —— 找"板块/抱团簇"**
- Louvain / Leiden（模块度最大化，最常用、快）
- 谱聚类（拉普拉斯矩阵特征向量）
- 层次聚类（适合 MST，输出树状图 dendrogram）
- 经验：检测出的社区常与官方行业分类（GICS / 申万）高度吻合但不完全一致——能发现"市场实际如何抱团"，包括跨行业主题联动

**(C) 网络拓扑指标 —— 刻画整体市场状态**
- 平均路径长度、聚类系数、网络密度、社区数量
- 危机时系统性变化：相关性飙升（"everything moves together"）→ 网络致密、集中、路径缩短。可作为**市场状态/危机的拓扑温度计**

### 2.3 三大应用的概念陷阱（本系统最核心的判断）

#### 应用一：抱团股票 —— 区分"结构聚类"与"真抱团"

社区检测任何时候都能给出一堆 community（同板块天然相关），那是**市场固有结构**，不是抱团。
真抱团（资金异常集中、内部联动远超基本面）必须看**动态量**：

- 簇内平均相关性（"温度"）随时间走高
- 社区致密化（密度、簇内边权上升）
- 与大盘/其他簇**解耦**：`簇内相关 ÷ 簇间相关` 比值飙升
- 资金虹吸指纹：其他簇松散化、该簇致密化

> **抱团指标定义**：滚动窗口下，某 community 的（内部平均相关性 + 密度 + 对全市场引力占比）**同时单调上升**。

#### 应用二：核心龙头 —— 区分两种"核心"

| 类型 | 工具 | 含义 |
|------|------|------|
| **结构核心**（对称） | degree / eigenvector / PageRank / betweenness | 最同步、最有代表性的票；介数高=跨板块枢纽 |
| **趋势引领者**（方向性） | 滞后相关 / 转移熵 / Granger 因果 | 先动、别人跟（时序+因果概念，对称相关测不出） |

> 一句话：**对称相关网络找"结构核心"，有向信息流网络找"趋势龙头"**。混用二者是最常见错误。
> 把方向性关系做成**有向网络**：出度信息流大者=引领者，入度大者=跟随者。龙头切换会在信息流方向上提前显现。

#### 应用三：趋势 —— 价值在"相变"而非"快照"

- **抱团生命周期**：形成 → 致密 → 见顶 → **瓦解**
- 最有交易价值的信号常在**瓦解端**：簇内相关性见顶回落、社区松散 = 趋势反转早期信号
- 全市场密度 = 风险温度计；社区合并/坍缩成单一巨簇 = 趋势末期或危机前兆

### 2.4 根本陷阱：反身性与滞后（决定能否盈利的分水岭）

- 相关性网络本质是**同步/滞后指标，不是领先指标**
- 网络清晰显示致密抱团簇时，交易很可能已拥挤、接近反转——**结构看得最清楚的时刻，往往是风险最大的时刻**
- 反直觉结论：**alpha 多在"瓦解端"而非"形成端"**

---

## 3. 结论

1. **抱团、龙头、趋势三件事用网络都能做**，但每件都有概念陷阱，识别陷阱比堆工具更重要。
2. **网络更适合做风险管理与择时的环境判断，而非直接选股 alpha 引擎。** 重心应放在监测拓扑拐点（相变），而非用静态网络选股。
3. **必须用动态（滚动窗口）网络**，静态快照会误导（相关结构强非平稳）。
4. **找龙头必须换成有向信息流工具**（转移熵 / 滞后相关），对称相关网络做不到。
5. **数据清洗占 80% 工作量**，算法本身最简单。幸存者偏差与 point-in-time 成分股是实盘前必须解决的第一问题。

## 4. 执行计划

### 4.1 数据需求（分层）

**第一层 · 最低限度（跑通核心）**
- 一篮子股票复权日线收盘价
- 股票池 100–500 只（建议直接用指数成分股）
- 时间长度满足 T/N ≥ 2（理想 ≥ 5）
- 数据源：yfinance / Stooq / Tushare（免费足够）

**第二层 · 动态网络 + 抱团温度**
- 更长历史（保证每个滚动窗口 T>N，否则必须 RMT/收缩去噪）
- 成交量 / 成交额（验证资金虹吸）
- 干净股票池：处理幸存者偏差、point-in-time 成分股、退市/并购/改名

**第三层 · 找龙头（有向网络）**
- 分钟级 / 小时级高频数据（日线测领先-滞后太粗）
- 注意微观结构噪声、非同步交易造成的虚假滞后相关
- 数据源：Polygon / Databento / 交易所直采

**第四层 · 区分真假抱团（关联属性）**
- 行业分类（GICS / 申万）、市值、流动性、基本面、资金流/持仓（13F、北向、ETF 申赎）

### 4.2 分阶段路线图（按优先级，先验证再加码）

**阶段 0 — 数据管线**
- yfinance 拉 100–500 只标普成分股、3–5 年日线
- 处理：复权、时间戳对齐、停牌缺失、剔除低流动性票
- 产出：干净的对数收益率矩阵

**阶段 1 — 静态网络（sanity check）**
- 相关矩阵 → RMT/收缩去噪 → Mantegna 距离 → MST + Louvain 社区 → 可视化
- 校验：检测社区 vs 官方行业分类的吻合度
- 最小可行流程：`收益率 → 相关矩阵 → Mantegna 距离 → MST → Louvain → 可视化`

**阶段 2 — 动态网络 + 抱团温度（核心验证）**
- 滚动窗口构建动态网络
- 计算"抱团温度"时间序列：簇内平均相关、密度、簇内/簇间相关比、引力占比
- 校验：与历史著名抱团行情（如茅指数/宁组合 2020–21）能否对上号

**阶段 3 — 有向信息流网络（找龙头）**
- 单独引入小批高频数据
- 滞后相关 / 转移熵 / Granger → 有向网络
- 校验：引领关系是否稳定、可预测；龙头切换是否提前显现

**阶段 4 — 事件研究（检验交易价值）**
- "抱团瓦解"信号出现后，簇内股票后续收益是否系统性走弱
- 这是判断方法有无 alpha 的关键一步

### 4.5 关键坑清单（编码时务必处理）

- **相关 ≠ 因果**：方向性分析必须用转移熵/Granger，不能用对称相关推断传染方向
- **非平稳**：务必滚动窗口，禁用静态网络下结论
- **窗口两难**：短窗口反应快但噪声大，长窗口稳定但滞后 → 多尺度并行
- **维度灾难**：N 接近 T 时相关矩阵不可靠 → RMT 去噪
- **幸存者偏差**：禁止用"今天还存在的票"回溯历史
- **高频微观结构**：非同步交易制造虚假滞后相关，需专门处理

## 附：核心公式速查

- Mantegna 距离：`d_ij = sqrt(2 * (1 - ρ_ij))`
- 抱团温度（概念式）：`Heat(C, t) = f(簇内平均相关↑, 密度↑, 簇内/簇间相关比↑, 引力占比↑)`，四者同时单调上升 → 真抱团
- 龙头判定（有向网络）：`出度信息流(node) 大 → 引领者；入度大 → 跟随者`
