# 20 周线交易系统 · 规则说明书

> 用途:个人趋势风控的固定规则,作为下一步程序化实现的依据。
> 核心原则:**慢趋势用均线管,快暴跌用硬止损管。两者并联,优先级不同。**

---

## 0. 设计前提(必须先想清楚)

| 项 | 说明 |
|---|---|
| 框架唯一性 | 一个标的只用**一套**框架做买卖决策。要么趋势(技术面),要么基本面,**不在执行时临时切换叙事**。 |
| 本说明书定位 | 这是一套**趋势风控系统**,用于控制回撤,不是基本面持仓逻辑。 |
| 最大失败模式 | 涨时讲技术面,该止损时改讲"基本面这么好不可能跌" → 两套纪律都丢失。代码层面应禁止这种混用。 |
| 入场 vs 退出 | **入场和退出是两份工作。** Layer 0–2 全是**退出/持仓风控**;入场逻辑见 §7,与本系统**配对但独立**——不要拿退出框架去否定入场问题,也不要拿退出的触发条件(周收盘站回)当唯一入场点。 |

---

## 1. 风控分层结构(优先级从高到低)

程序应按以下顺序逐层判断,**高优先级层一旦触发,直接接管,跳过低优先级层**。

```
Layer 0    熔断 / 硬止损      ← 最高优先级,处理极端与不可逆事件
Layer 1    事件驱动检查        ← 处理基本面突变
Layer 1.5  初始技术止损 / 保本线 ← 入场 setup 的止损持续生效(日收盘判),补层间缝隙
Layer 2    20 周线趋势规则     ← 处理常态趋势反转(默认主逻辑,只看周收盘)
```

### 1.1 Layer 1.5 — 初始技术止损持续生效(2026-07 补)

**它补的缝**:突破买点常在 20 周线上方 20–30%。此后阴跌——每天 −5% 连续四天(不触发 Layer 1 单日 −20%),5 日峰谷 −22%(不触发 Layer 0 −30%),又远在 20 周线上方——三层全静默,亏损已达数 R。同时 sizing 按 setup 技术止损算 1R,若入场后该止损被丢弃,"1R"只是入场仪式。

**规则**:
- 成交当天把 setup 的止损价登记进 `select.yml US_SWING_STOPS`;离场后删除。
- **有效止损 = max(初始技术止损, 保本线(浮盈≥+30% 后 armed = 成本价), 20 周线)**。
- 初始止损/保本线按**日收盘**判定(与 ADR-0002 一致),跌破 → 今日离场,不等周五;20 周线仍只按周收盘判定。
- 组合层:**OPEN HEAT**(全部持仓有效止损同日打穿的总损失)≤ 6% 权益——池内高相关,5 个名义仓位 ≈ 1 个主题仓位,风险预算做在主题层。

## 2. Layer 2 — 20 周线趋势规则(常态)

### 2.1 判定基准

- **只看周收盘(weekly close),忽略盘中和单日波动。**
- 盘中跌破 / 单日反弹 → 不构成任何买卖信号,**不触发决策**。
- 真正的判断时刻 = **每周五(或当周最后交易日)收盘**。

### 2.2 触发条件(可配置过滤器)

| 参数 | 默认值 | 说明 |
|---|---|---|
| `confirm_on` | `weekly_close` | 只认周收盘 |
| `buffer_pct` | `3% ~ 5%` | 跌破幅度需超过此缓冲才算有效跌破,过滤毛刺 |
| `consecutive_weeks` | `1`(可设 `2`) | 连续 N 周收在线下才确认,降低 whipsaw |
| `use_ma_slope` | `true` | 结合均线斜率判断含义(见 2.3) |

### 2.3 斜率修正(同样是跌破,含义不同)

- 20WMA **仍上行** + 跌破 → 较可能是回踩,可保守(分批/观望)。
- 20WMA **走平或掉头向下** + 跌破 → 趋势转弱信号强,执行减/清。

### 2.4 执行方式(避免非黑即白)

- 确认有效跌破 → **先减一部分**(例:50%)。
- 后续周收盘**站回线上** → 加回。
- 不要单次全清 / 全进,降低被甩成本。

### 2.5 已知固有成本

- **Whipsaw(被甩耳光)**:卖在阶段底部、随后涨回 = 趋势跟踪系统的**固有成本,不是规则失灵**。
- 接受这套规则 = 接受它会偶尔让你难受。
- 若觉得过于敏感 → 调 `buffer_pct` / `consecutive_weeks`,**不要靠临时找例外来推翻系统**。

## 3. Layer 1 — 事件驱动检查

当出现**远超历史正常波动**的下跌时,先于周线规则触发:

- 触发阈值示例:单日或数日内 `< -20%`(可按标的历史波动率自适应)。
- 动作:**立即去查原因**,不等周五。
- 原因分类:
  - 基本面**不可逆**(造假 / 资不抵债 / 退市 / 暴雷 / 行业塌方)→ 不等任何确认,**直接处理**。
  - 原因不明、流动性仍在 → **先减仓控风险**,保留一部分等信息明朗。

---

## 4. Layer 0 — 熔断 / 硬止损(最高优先级)

> 对应"暴跌 80%"这类问题:**绝不死等周五。**

| 规则 | 说明 |
|---|---|
| 独立于趋势系统 | 这是一条与 20 周线**并联**的硬开关,优先级最高。 |
| 触发即接管 | 一旦触发,跳过 Layer 1/2 的所有"确认"逻辑。 |
| 适用场景 | 崩塌式下跌,本身已是结论,不需要均线再去"确认趋势坏了"。 |
| 风险提示 | 暴跌 80% 常伴随后续归零 / 长期停牌,**可能根本跑不掉** → 处理要快。 |

### 关键认知

> 真到了 -80% 的位置,"现在跑 vs 周五跑"的差别,远小于"当初有没有在第一道风控线就出来"。
> -80% 意味着**前面一定错过了更早、更温和的离场信号**。
> 教训不在结尾,而在于**止损体系要分层**:极端事件用硬止损接管,常态才交给 20 周线。

## 5. 程序化伪逻辑

```python
def decide(position, price_data, fundamentals):
    # Layer 0: 熔断 / 硬止损(最高优先级)
    if crash_detected(price_data, threshold=HARD_STOP_PCT):
        return ACT_NOW("hard_stop")          # 立即处理,不等周五

    # Layer 1: 事件驱动
    if abnormal_drop(price_data, threshold=EVENT_DROP_PCT):
        cause = investigate(fundamentals)
        if cause.is_irreversible:
            return ACT_NOW("fundamental_break")
        else:
            return REDUCE_PARTIAL("uncertain_cause")  # 先减仓,留观察

    # Layer 2: 20 周线趋势(默认,仅在周收盘评估)
    if not is_weekly_close():
        return HOLD("not_decision_time")     # 盘中/单日波动 → 不动

    if confirmed_break_below_20wma(
        buffer=BUFFER_PCT,
        weeks=CONSECUTIVE_WEEKS,
        slope=ma_slope()
    ):
        return REDUCE_PARTIAL("trend_break")  # 先减,站回再加
    elif reclaimed_above_20wma():
        return ADD_BACK("trend_reclaim")

    return HOLD("trend_intact")
```

## 6. 一句话总结

> **20 周线管"温水",硬止损管"开水"。**
> 慢的那套(均线)不去处理快的事(暴跌);该执行时就按周收盘执行,不被单日反弹牵着走。

## 7. 入场框架(超跌优质·放量转折)— 与退出分工

> 本节是**入场**逻辑,与 Layer 0–2 的**退出/风控**配对但独立。
> 适用对象:**仅优质标的**(`t_us_undervalue.py` 超跌优质池)。对随机/低质标的,"底部放量"= 赌均值回归 = 讨债型信号,**不适用**。
>
> 🛠 **实盘脚本:`t_us_bottom_entry.py`** —— 扫当前 SP500∪NDX,列出此刻"超跌(距252日高≤−30%)+ 跌破20周线"的入场候选,按 SEC PIT ROE 分 PASS/FAIL/UNKNOWN 层(质量=防归零分层,非硬门),报告自带本节纪律。回测验证见 `t_us_bottom_entry_backtest.py`(§7.5–7.6)。

### 7.1 为什么对优质标的成立(而对垃圾股不成立)

- **回报的 horizon 结构支持它**:~1月反转 / 3–12月动量 / **3–5年反转**。优质股深跌后回归,吃的是长周期反转 + 质量溢价,**不是**赌"市场欠一个反弹"。
- **"底部"性质不同**:垃圾股下跌多为**永久性减值**(回不来);优质股深跌多为**暂时性错杀**(系统性抛售 / 板块轮动 / 一次性 miss),可恢复。
- **放量是关键过滤器**:放量上涨 = 底部有人接筹(accumulation / 抛售高潮后翻红),是区分"真转折"与"死猫跳"的那个变量。缩量反弹大概率是下跌中继。
- **抓的是收益集中的那几根爆发日**:复苏第一段放量上涨往往就是爆发日起点;等站回 20 周线才进,最肥的一段已走完。

### 7.2 三个前置条件(edge 全在这里)

| 条件 | 说明 | 不满足则 |
|---|---|---|
| **质量为真** | 用 decomposition 判:这波跌是**估值压缩(multiple,可恢复)** 还是 **盈利减值(EPS-driven,可能不可逆)**?排除基本面 break(= Layer 1 检查)。**质量是【左尾保护/防归零】,不是【收益放大器】**(见 §7.6)。 | 价值陷阱,**别接** |
| **探针式分批** | 抓不到精确底,底部常横盘筑底数周到数月。第一根放量**只建底仓(probe)**。 | 一把梭 = 接刀风险 |
| **放量须是衰竭型** | 长期下跌后的衰竭位、形成抬高的低点、上涨日量能持续压过下跌日、次日站得住。 | 下跌中继的反抽,**陷阱** |

> ⚠️ **回测校准(见 §7.5):机械"放量"过滤器不贡献 EV,甚至略拖累。** 放量的价值(若有)只在人工区分"衰竭吸筹 vs 新闻暴力反弹",**不要把"量≥N×均量"当 go-signal**。edge 来自"超跌优质 + 线下"这个状态本身,不是来自量。

### 7.3 入场链(entry → 持仓交还退出系统)

```
1. 池:t_us_undervalue.py 超跌优质(质量已过滤)
2. 触发:底部放量转折(衰竭型放量上涨,§7.2 第三条)
3. 动作:建底仓 probe(非满仓)
4. 加仓点 A:抬高的低点成立(higher-low)
5. 加仓点 B:周收盘【站回 20 周线之上】→ 趋势确认,加
6. 进场后:持仓交还 Layer 0–2 管理(止损 / 事件 / 周线减清)
```

> **入场用 §7,持仓用 Layer 0–2。** 站回 20 周线在退出系统里是"加回"信号(§2.4),在入场链里是**加仓点 B**——不是初始入场点。两类标的入场点不同:动量/突破型 = 站上确认后进;超跌优质 = 放量转折先进、站回再加。
>
> ⚠️ **回测校准(§7.5):站回入场比线下入场更差**(等站回最便宜那段已走完)。所以**初始仓越早越好(线下就建)**;"站回"是**确认 + 加码**的点,不要把它当唯一入场点,更不要等它。

### 7.4 已知失败模式

- **把结构性受损误判成超跌优质** → 价值陷阱,本框架最大失血点。质量验证不是脚注,是全部 edge。
- **把单日放量当全清信号** → §7.2 要求探针 + 确认,不要一根 K 线定生死。

### 7.5 回测校准(2026-06-29 事件研究)

> 池=当前 SP500∪NDX 513 只 × 15 年日线(yfinance);事件 = 距252日高≤−30% 且收盘<20周线;前向窗口 21/63/126/252 日;基准 = 同日 QQQ。
> **告警一:幸存者偏差**——池只含活下来的票,**专门抹掉归零的真·价值陷阱**,故绝对收益偏乐观、陷阱率是**下限**;但组间相对比较不受影响(各组等量受偏)。
> **告警二:事件跨标的/时间重叠**,只读 EV(均值/中位/胜率),不做显著性检验。

实测结论(126 日窗口):

| 入场条件 | n | 中位% | 胜率% | 陷阱% | 跑赢QQQ% |
|---|---|---|---|---|---|
| 只超跌+线下(BASE) | 7554 | **13.5** | **69.8** | **30.2** | 49.7 |
| +缩量反弹 | 6053 | 13.2 | 69.0 | 31.0 | 50.4 |
| +放量≥1.5× | 2581 | 11.9 | 67.1 | 32.9 | 48.9 |
| +放量≥2.5× | 468 | 11.8 | 66.9 | 33.1 | 53.4 |
| 站回线上(reclaim) | 3933 | 11.5 | 68.5 | 31.5 | 46.6 |

**三条钉死的结论:**
1. **edge 在"超跌优质 + 线下"状态本身**,且**越早进越好**(线下 > 站回)。
2. **机械"放量"过滤不贡献 EV、略拖累**(中位/胜率下降、陷阱率上升)。别把放量当 go-signal。
3. **alpha 靠右尾少数大复苏**(全程跑赢 QQQ 仅 ~47–53%,中位仅打平)→ **必须分散持多只 + 拿得住**,单押一只≈打平 QQQ。

> 脚本留档:`scratchpad/bt_bottom_vol.py`(主)+ `bt2_reclaim.py`(站回补测)。

### 7.6 PIT 质量重测(2026-06-29 · SEC EDGAR)

> 把 §7.5 的"成分股代理质量"换成**真 point-in-time ROE**:SEC EDGAR XBRL,ROE_FY = NetIncomeLoss / StockholdersEquity 按 fiscal-year-end 撮合,**只用 `filed ≤ 信号日` 的 10-K**(取首次申报值,防 restatement 前视),近≤3 年均值 >12% = 质量 PASS。覆盖 483/512(94%;外国 20-F 发行人等剔除)。把 BASE(超跌+线下)7554 事件按 PIT 质量拆桶。

| 桶 | n | 252d中位% | 252d alpha% | 跌>40%率 |
|---|---|---|---|---|
| Q=PASS(ROE>12%) | 3138 | 27.3 | 5.3 | **2.5** |
| Q=FAIL(ROE≤12%) | 2943 | **30.5** | **16.1** | 4.8 |
| Q=UNKNOWN | 1473 | 24.6 | 2.4 | 5.0 |

**反直觉但有解释的结论:**
1. **论 raw 反弹,质量过滤选中的是更差的一半**——低/负 ROE 的超跌股反弹**更猛**(高 beta 弹性 + 多重压缩待修复)。
2. **但这是幸存者偏差的陷阱,且不对称**:池=当前成分,低 ROE 里**破产/退市/归零的真·价值陷阱被整组删掉**,FAIL = "事后已知活下来的垃圾";高质组几乎不退市,不受此删。**本回测先天无法给质量记功**(它防的归零根本不在样本)。
3. **左尾证据救回质量**:即便在幸存者内部,PASS 的**深跌(>40%)率约为 FAIL 的一半(2.5% vs 4.8%)**,再叠加 censored 的归零 → 质量是明确的**左尾保护**。
4. **定论:质量过滤的 job 是【价值陷阱过滤器/防归零】,不是【收益放大器】。** 与 20 周线系统"回撤控制"定位一致;"对优质标的"成立,理由是**防归零**而非**博更大反弹**。
5. **天花板**:要真正量化质量的护身价值,需**无幸存者偏差票池**(历史成分 + 退市票含归零段);yfinance/当前成分做不到,本结论到此为止。

> 脚本留档:`scratchpad/build_pit_quality.py`(SEC EDGAR PIT 质量库)+ `bt3_pit.py`(质量拆桶)+ `bt3b_tail.py`(左尾)。
