# 指标与策略设计心法:GPS,不是讨债单

> 一句话:**好的指标是 GPS(我现在在哪),坏的指标是讨债单(市场欠我一个反向运动)。**
> 设计每一个信号时永远问一句——它是在**定位**,还是在**记账**?

本文档源自一次对"股票分析应该只看眼前、不看过去"这一说法的拆解,落地为本项目(US swing 系统)在设计指标、回看窗口、过滤条件时的纪律。

---

## 1. 缘起:拆解"只看眼前,不看过去"

原命题:*"一个股票已经涨了一年,这和它未来走势没有任何关系,所以当下分析不该考虑它一年内的走势。"*

这句话把一个**值得保留的纠偏**和一个**错误的认识论**捆在了一起。

### 对的一半:价格是 path-independent 的
股票不记得你的成本,过去涨幅本身不构成未来涨跌的"理由"。这条只反对两个具体错误:

- **Anchoring 到成本/已实现涨幅**:"已经翻倍了所以不该再涨"、"涨这么多了该止盈"。
- **Naive mean-reversion**:"涨了一年所以该跌了"。

第二条恰恰是本项目 backtest 推翻过的:**所有 overbought 信号后面都是继续涨,不是反转**。在这个范围内,"别让历史涨幅绑架 forward judgment"是对的。

### 错的一半:混淆了两种"关系"
说话者把下面两件事偷换成了同一件:

| 关系 | 强度 | 说明 |
|---|---|---|
| 因果路径依赖:过去价格 → 未来价格(机械因果) | 弱/几乎不存在 | "涨了"本身不机械地决定"跌不跌" |
| 信息含量:过去价格 → 当前状态的信息 → 未来期望收益 | **强** | 过去是关于"你现在站在哪"的证据 |

"涨了一年和未来没有任何关系"当作机械因果是对的,当作**信息无关**是错的。

### 实证上"没有任何关系"是假的(horizon-dependent)
过去收益与未来收益的关系**取决于时间尺度**,不是零:

- **短周期(~1 个月)** → reversal(反转)
- **中周期(3–12 个月)** → momentum,正自相关(Jegadeesh-Titman)
- **长周期(3–5 年)** → mean reversion(DeBondt-Thaler)

"涨了一年"正卡在 momentum 尾部、reversal 开始渗入的边界——正确反应是**关注 exhaustion 信号**,而不是"无视历史"。

### "只看眼前"本身不自洽
你无法在不引用历史的前提下定义"眼前":当前 P/E 高低只有相对历史才有意义;ATH、超买、估值分位、MA200——全是历史的函数。本系统的止损挂在 MA200,那就是一段历史的均值。**剥掉过去,"当下分析"连坐标系都没有了。**

### 精炼后的正确版本
> 不要让价格图的历史涨幅 **anchoring** 你的 forward valuation;
> 但必须用**基本面的历史 + 这段涨幅的归因分解**来定位现在。
>
> **看过去,是为了定位现在,而不是为了给未来记账。**

## 2. 七条设计纪律

核心一句:**让指标去测"我现在在什么状态",而不是去赌"市场欠我一个反向运动"。** 以下全部从这条派生。

### 2.1 先给每个指标贴标签:测"状态"还是测"债务"
- **状态型**:趋势还在不在?波动率高不高?cycle 在哪个阶段?(价格 vs MA200、ADX、vol regime、RS 分位)
- **债务型**:暗含"涨太多该回头"(RSI 超买就卖、偏离均线太远就反向)。

债务型 = "市场欠我一个回调"的思路,在趋势里是**负 EV**——这正是本项目 backtest 出来的(MACD / RSI / EMA crossover 在单边牛市亏钱)。
**纪律:每个信号都明确归类,别在趋势行情里悄悄用了一个 mean-reversion 信号。**

### 2.2 把"过去"编码成相对位置,不要编码成绝对涨幅
- ❌ "涨了 X%" → 债务思路,不可用。
- ✅ "处在过去一年的第几分位 / 离 MA200 多远 / RS 排全市场前百分之几" → 状态思路,可用。

本系统已经在做对的事:**RS Rating、MA200 止损**,本质都是"拿历史当尺子量当下位置",而不是"算市场欠多少"。

### 2.3 Lookback 窗口 = 你押注哪个机制
过去与未来的关系 horizon-dependent(见 §1):1 月反转、3–12 月动量、3–5 年反转。
**你选的回看窗口本身就是一个押注**;12 月动量和 1 月反转押注的是相反的力。
**纪律:窗口从机制出发选,不是从"哪个数 backtest 最漂亮"选(后者是 overfitting)。能讲清"为什么这个窗口有效"再用。**

### 2.4 最有价值的应用:用 regime 做条件过滤
同一信号在不同状态下 EV 完全不同(本项目发现:**超买在牛市里从不反转**)。威力最大的不是单一指标,而是**分层**:

1. 先用"过去"判断 regime(MA200 斜率、vol regime);
2. 再决定"当下信任哪类信号":
   - **趋势 regime** → 只信 momentum / 突破,**关掉**反转信号;
   - **震荡 regime** → 才放反转信号进来。

这就是"用过去定位现在,再行动"最直接的落地,比堆一堆指标有用得多。

### 2.5 基本面用 decomposition,不用 level(cyclical 会反转)
别直接喂原始价格或原始 EPS,要**拆**:这波是 **EPS-driven** 还是 **multiple-driven**?
对 cyclical(如 MU),"低 P/E + 高 earnings"在**顶部是卖出信号**,naive value 指标会指反方向。
**纪律:基本面指标必须内建 cycle-stage 意识(P/E 分位与 EPS 轨迹分开追踪)。**

### 2.6 诚实性:只能用"当时知道的过去"
"过去是关于状态的信息"只在那信息**当时真的可得**时才成立。
**Backtest 必须 point-in-time:no restatement、no survivorship、no look-ahead。** 否则不是在测状态,是在作弊。

### 2.7 Binary event 单独切出来
日历也是"过去信息":知道财报日,就该把**连续趋势暴露**和**离散事件风险**分开处理。
趋势策略不该闭着眼睛穿过财报(如 MU 6/24 FQ3)。
**趋势信号管 continuous exposure,事件用 reduce / collar 单独管。**

## 3. 与本项目已有 edge 的呼应

这套心法不是空谈,本系统的现有 edge 全部建立在"用过去定位现在"之上:

- **QQQ / ATH 买入 ~75% 胜率**、**RS Rating 全市场分位筛选** → 全是"用过去走势"。若过去真和未来无关,这些 edge 不可能存在。
- **MACD / RSI / EMA crossover 在单边牛市亏钱** → 债务型信号在趋势 regime 里负 EV 的实证。
- **MA200 止损** → 把"过去"编码成相对位置(§2.2),而非绝对涨幅。
- **MU 6/24 FQ3 作为 binary event** → §2.7 的现实场景:过去一年是衡量"什么已被 price in、guidance 要超多少才算 beat"的基线。

## 4. 一页速查

| 问题 | 正确做法 |
|---|---|
| 这个信号在干嘛? | 定位(GPS),不是记账(讨债单) |
| 怎么编码"过去"? | 相对位置/分位,不是绝对涨幅 |
| 回看窗口选多长? | 从机制选(反转/动量/长反转),能讲清才用 |
| 同一信号到处用? | 先判 regime,再决定信哪类信号 |
| 基本面怎么喂? | decompose(EPS vs multiple),带 cycle-stage |
| Backtest 可信吗? | point-in-time,no look-ahead/survivorship |
| 趋势穿越财报? | 不。事件风险单独 reduce/collar |

> **看过去 ≠ 猜未来该涨该跌;看过去 = 搞清楚"现在到底是个什么情况"。**
