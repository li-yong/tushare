# 从新闻事件识别股价见顶:压倒骆驼的最后一根稻草

> 一句话:**顶不是某一根稻草压出来的,是"新闻主语的迁移"——当新闻的主语从"公司自己"切换成"生态系统",而且这个切换发生在基本面最好的那一天,就是顶部区。**

本文档源自 MU (Micron) 2026-06 见顶的完整复盘。目标是把"新闻→见顶"从事后诸葛亮变成一套可核对的状态测量(GPS,不是讨债单,见 [indicator_design_state_vs_debt.md](indicator_design_state_vs_debt.md)),并为 20 周趋势系统的 L1 事件层([twenty_week_trend_system.md](twenty_week_trend_system.md))提供触发词表的依据。

## 2. 稻草的分类学:四类顶部新闻各测什么

每条新闻是对**周期所处状态**的一次测量,合起来才构成见顶判定。单看任何一条都不构成做空/清仓依据(那就成讨债单了)。

### 2.1 利好出尽 + 关键K线(价格侧必要条件)

本周期最好的消息(超预期财报+超预期指引)与 climax bar 同日出现:跳空创新高、收在开盘价之下、长上影、天量。当"最好的新闻"买不出"守得住的新高",说明所有想买的人已经买完。

- **二者缺一不可**:没有超级利好的长上影只是震仓;没有 climax bar 的超级利好只是又一次上台阶。
- 这条只能由价格测出,新闻测不出。对应 [key_kline_methodology.md](key_kline_methodology.md) 的关键K线判定。

### 2.2 监管/诉讼(周期温度计爆表)

反垄断诉讼本身几乎从不直接杀股价(06-25 当天照样 +15.7%)。它的信号价值是**历史定位**:DRAM 反垄断案从来出现在价格周期顶部附近——2002 年 DOJ 刑事案、2018 年集体诉讼、2026-06-25 本案,无一例外。

诉讼 = 涨价已经激烈到社会/政治不可持续。它测的不是这家公司,是**周期温度**。

### 2.3 客户公开反目(需求侧开始反噬)

Apple 涨价 20% 并公开点名供应商,Micron 反呛客户——供应链上下游从"谈判桌内博弈"升级为"公开互撕",是周期顶的经典行为学特征:

- 客户把成本转嫁消费者 → **需求破坏启动**(价格弹性开始起作用);
- 客户同时在加速寻找替代、调整库存、砍单——公开抱怨是冰山露出水面的部分。

### 2.4 下游承认过剩(叙事终结,真正的最后一根稻草)

前三根稻草都是"涨价的副作用",逻辑上还能用"供不应求"圆回来。但 HBM 的边际大买家(Meta)亲口承认 "excess compute capacity",直接击穿整个叙事的地基——"AI 需求无限"。叙事死了,-10.6% 只是记账。

**最后一根稻草的判别特征:它不攻击"公司",它攻击"需求假设"。** 反垄断可以辩护("我们合法竞争"),客户抱怨可以辩护("是他们当年压价"),但"买家说自己买多了"无法辩护。

注意主语与受害者的分离:这条新闻对 Meta 自己是利好(+10%),杀的是整条算力供给链(neocloud -12%、MU -10.6%)。**扫描时要按"谁的需求假设被击穿"归因,不能按新闻主语归因。**

## 3. 可泛化的见顶模式(checklist)

按出现顺序:

1. **新闻主语迁移**:见顶前新闻主语是公司自己(财报、产品、订单、新高);顶部区新闻主语变成生态(客户、监管、法院、下游买家)。主语切换比任何单条内容更重要。
2. **"最好消息日"就是最危险的日子**:超预期利好 + climax bar 同日 → 顶部候选确立。
3. **上下游公开冲突 / 成本转嫁见报** → 需求破坏启动。
4. **监管/反垄断立案** → 周期温度计爆表(辅助定位,不是触发)。
5. **下游最大买家承认过剩** → 叙事终结,最后一根稻草。
6. **确认在弹不动**:顶后第一次板块反弹,该股相对强度掉队(MU +1% vs SOXX +4%)→ 死亡确认,此后反弹是出货机会不是回场机会。

纪律边界(防讨债单化):

- 1–5 都是**状态测量**,不是反向下注的理由;仓位动作只在价格确认(6,或 L1/L1.5/L2 止损触发)后执行。
- 主语迁移只对**抛物线末段/周期股**有分辨力;慢牛里客户抱怨、诉讼常年有,不构成信号。前置条件:股价处于加速段(可用 distance-to-20wk 或近月斜率界定)。
- 样本量诚实:本文档 n=1(MU),模式与 2002/2018 DRAM 周期、历史大宗周期顶吻合,但未做系统回测。每新增一个案例回来补一节。

## 4. 与现有系统的对接

**已程序化(2026-07-03): `t_us_news_top_detector.py`。** 分层实现与验证状态:

- **新闻侧**:`grok_lib.top_scan`(结构化 JSON schema:四类稻草+主语迁移+按"需求假设受害者"归因,非关键词匹配);稻草类型字符串定义在 `constant.py`。

- **§3 边界**:非加速段(close 距 20 周代理线 <20% 且无新鲜 climax)时新闻侧风险封顶 low——慢牛里诉讼/客户抱怨常年有。
- **L1 语义**([twenty_week_trend_system.md](twenty_week_trend_system.md)):action 最高只到 "de-risk-pending-price-confirmation",仓位动作仍归价格系统(L0/L1.5/L2)。

- **已接 cron(2026-07-03)**:`us_weekly_run.sh` 周日档,排在 signal-attrib 之前(当周稻草 episode 先落账再归因);14 天窗口 × 7 天节奏刻意重叠,持续存在的稻草刷新同一 episode 而非新开。~$4/次。

## 6. 镜像:从新闻事件识别启动(火种)

见顶方法论逐层取镜像,得到启动检测。核心命题的镜像:**底不是某一根火种点出来的,是"新闻主语的回归"——底部区新闻主语是行业性阴霾(过剩、砍单、宏观),当主语第一次迁移回"公司自己"(订单、产品、提价),且价格同窗打出进场型关键K线,就是启动区。**

### 6.1 四类火种(稻草的镜像)

| # | 火种 | 镜像自 | 测什么状态 |
|---|---|---|---|
| 1 | `catalyst_breakout` 催化剂+价格半 | §2.1 利好出尽+climax | 本周期级别催化剂(财报大超+上修/重磅订单/design win)**且**同窗价格半成立。**两半缺一不可**:没有价格半的催化剂只是又一条新闻;没有催化剂的突破只是技术行为。价格半有两种形态(§6.1b)。 |
| 2 | `supply_destruction` 供给侧出清 | §2.2 监管温度计爆表 | 行业性减产/砍资本开支/产能退出/破产 = 周期温度计**冰点**(DRAM 减产新闻从来出现在价格周期底部附近)。辅助定位,不是触发。 |
| 3 | `demand_commitment` 需求公开承诺 | §2.3 客户公开反目 | 大客户公开签约/长约/生态伙伴公开站台——需求侧从"谈判桌内博弈"升级为"公开背书"。 |
| 4 | `shortage_admission` 下游承认短缺 | §2.4 下游承认过剩 | 下游边际买家承认"买不到/接受提价"——新叙事(需求假设)确立的第一根火种。归因按**"谁的需求假设正在建立"**(demand_builder),不按新闻主语。 |

外加布尔量**主语回归**(`subject_birth`,镜像主语迁移):新闻主语是否已从行业阴霾/宏观迁移回公司自己。

### 6.1b 价格半的两种形态:顺势确认 与 利好错杀背离

§2.1 的本质是**背离**——最好的新闻买不出守得住的新高。所以启动侧的价格半不能只有顺势确认,真正的镜像也是背离:

1. **顺势半**:新鲜且止损未破的进场型关键K线(BREAKOUT/FIRST_KISS/POCKET_PIVOT/EARNINGS_GAP,key_kline 的 ★ 条件)。右侧、确认后,代价是可能明显偏晚。
2. **背离半(`SHOCK_DIVERGENCE` 利好错杀)**:**最强的新闻砸出深坑**——催化剂新闻同窗,价格却快速深跌:距 60 日高回撤 ≥20%、该高点距今 ≤15 bar(快崩是错杀,阴跌 20% 是下降趋势)、且 close 仍在 MA150 上(长趋势未破=质量防线)。止损 = 急跌段最低(跌破 = 还在自由落体,态失效),**按日收盘判**(ADR-0002)——V 底次日盘中下探破段低是常态(MU 案例 3/31 盘中 -2.2% 穿透后收回),盘中判会被洗出去。无量能门槛、不等企稳(bottom-entry 实证:放量无用、越早进越好)。

参考案例 MU 2026-03:03-18 史诗财报+首份 5 年 SCA,8 bar 内 471→322(-31.7%)但仍在 MA150 上。顺势半的第一个 ★ 在 04-22(收 487,距底 +51%);pullback_shock 被自家"距 252 日高 ≤15%"趋势门槛挡住;唯一同窗看见启动的组合 = 火种新闻 × 背离态,3/30 当日止损 318.4、1R 仅 1.1%。

### 6.2 纪律边界(镜像 §3,防追高化)

- **漏斗方向与见顶相反**:见顶是新闻先行(宇宙=持仓+观察池,天然小);启动的价值在池外发现,宇宙是 SP500∪NDX(~600 只),必须**价格先行**——先用免费的价格半筛候选(顺势 ★ 且未延伸,或背离态;背离态稀有,排最前、不会被 topN 挤掉),再对前 N 名做新闻侧验证(控制 Grok 成本)。
- **非扩张段前置条件**(镜像加速段前置条件):催化剂落在已延伸的票上(close 距 20 周代理线 ≥20%)不是启动信号,是追高陷阱——实证依据:弱势股/延伸段的单日大涨是陷阱(dip-buy vs pop-chase 研究)。此时新闻侧 launch_level 封顶 low。
- 无价格半时封顶 medium:火种未获价格确认,只配观察。
- **背离态只豁免"无价格半"封顶,不豁免"已延伸"封顶**:背离态本身就是价格半,但已延伸(dist20w ≥20%)的深坑是**见顶案例的第一波下跌**,不是错杀——此时新闻必然还金光闪闪(利好出尽的定义)。n=2 分离:MU 03-30(dist20w -3.5%,同日见顶侧=none → 真启动,+178%)vs MU 07-03(dist20w +54.8%,同日见顶侧=high ☠ → climax 后崩跌)。漏斗同理:已延伸的名字连 Grok 槽位都不给,见顶扫描负责它们。
- 全部是**状态测量**;进场动作仍走价格系统(两种价格半各给 entry/stop/1R,三层确认后手工执行),news 侧最高只到 "entry-candidate-pending-price-system"。
- 样本量诚实:启动侧 **n=1**(MU 2026-03-30,point-in-time 验证,见 §6.4);分类学是从见顶四稻草推的镜像。每新增一个案例回来补一行。

### 6.3 与现有系统的对接

- **新闻侧**:`grok_lib.launch_scan`(镜像 top_scan:四类火种+主语回归+按"需求假设建立者"归因);火种类型字符串在 `constant.py`(`SPARK_*`)。
- **价格侧**:顺势半复用 `t_us_key_kline` 的进场检测器(BREAKOUT/FIRST_KISS/POCKET_PIVOT/EARNINGS_GAP,自带 stop)+ `compute_status` 的 fresh/alive 判定;背离半 `_shock_divergence`(SHOCK_DD=20%/SHOCK_WINDOW=15 bar/MA150 之上,stop=急跌段最低)。漏斗复用其 `_bulk_ohlcv` + SP500∪NDX 宇宙。
- **脚本**:`t_us_news_top_detector.py` 同一脚本双向(`--top-only`/`--launch-only` 限向;限向跑记 tagged,不写 ledger 不占当日守卫);`--launch-universe`/`--launch-topn` 控制漏斗。
- **信号归因**:火种命中写 `signal_ledger`,source=`news_launch`(与 `news_top` 分开归因),仅 live 全量跑写入。
- **成本**:漏斗 top N 默认 12 → 每次 ~$1.2,叠加在 weekly 见顶扫描 ~$4 之上。
