# 潮·浪·風:市场的海洋心法

> 一句话:**浪是你交易的对象,風是你判断的对象,潮是你服从的对象。**
> 三层各有各的仪器——用错仪器(拿浪高测潮位)比不用仪器更糟。

本文档源自一次比喻推演(呼吸 → 海浪 → 潮浪風三分),落地为本项目(US swing 系统)看待"价格波动 / 叙事情绪 / 流动性周期"三个时间尺度的统一框架。定位是**认识框架,提示不门控**——它不新增任何规则,只解释既有规则各自站在哪一层(闭环登记: [rule_registry.md](rule_registry.md) #20)。姊妹篇:[indicator_design_state_vs_debt.md](indicator_design_state_vs_debt.md)(指标是 GPS 不是讨债单)。

## 1. 缘起:海水没有增多,浪却一个接一个

物理事实:海浪里的水分子几乎不位移,只是原地画圈;向前传播的是**波形和能量**,不是水。

这直指市场里最常见的认知错误——**"数水思维"**:"钱流入了股市所以涨了"。

- 每一笔买入都对应一笔卖出。钱从来没有"进入"股市,只是**穿过**它,从买方口袋换到卖方口袋。"资金流入"是财经媒体最误导的词汇。
- 一天可能只有 2% 的流通盘换手,但这个**边际成交价**给全部存量重新定价——万亿市值蒸发或凭空出现,没有一分钱真的消失或诞生。
- 所以:**价格是边际现象,浪高是能量现象,都不是"水量"现象。** 真正变化的是能量:情绪、换手意愿、杠杆。

但比喻需要一处关键修正(见 §4.1):海水总量其实**会**变,只是慢。短期波动是能量现象,长期趋势才是水量现象。潮就是那个慢变的水量。

## 2. 三层映射

| 海 | 市场本体 | 驱动性质 | 本系统的测量仪 |
|---|---|---|---|
| **潮** | 估值水位的慢变化:流动性周期(央行扩表/缩表、利率)+ 盈利总量的增减 | 引力级——外生、缓慢、不理会任何个股;**水量现象** | `t_us_regime_monitor`、20周线、market state |
| **風** | 叙事与情绪的能量注入:财报、Fed 表态、板块故事、AI capex 这类主题 | 把能量传给水面的东西——本身不是价格,但制造价格波动;**能量的源头** | `t_us_news_top_detector`、Grok 情绪层、`t_us_sector_rotation`、`t_us_cycle_monitor` |
| **浪** | 价格波动本身:日线 K 线、跳空、TR 放大、回调与反弹 | 被动结果——浪不驱动任何东西,它只是能量到达后的形状;**能量的形状** | `t_us_tech_swing`、`t_us_key_kline`、`t_us_gap_scan`、`t_us_tr_surge` 等全部价格扫描器 |

### 2.1 潮是水量现象,風浪是能量现象

潮的判断要看水量指标(流动性、盈利总量),不能用浪的形状去猜潮。**用日线形态判断牛熊,就是拿浪高测潮位——尺度错配。** 一个浪打得再高不改变潮向;退潮时最大的浪也把船留在更低的水位。

### 2.2 風分两种时间尺度

- **局地阵风** = 单日新闻、单份财报:制造一两个浪就散了,只值得浪级响应。
- **信风/季风** = AI capex、DRAM 周期这类持续数季的叙事:连续不断把能量泵进同一片海域,才值得仓位级响应。

`t_us_sector_rotation` 找的是"哪片海正在被季风吹",`t_us_cycle_monitor` 盯的是季风本身还在不在。**把阵风当季风追,是新闻驱动交易的典型死法**(参见 [news_driven_top_detection.md](news_driven_top_detection.md):tier-1 叙事见顶 vs 一次性催化剂)。

### 2.3 浪是唯一能直接观察的,但信息量最低

所有价格扫描器都在读浪形。浪形的价值不在预测下一个浪,而在**反推**風和潮:

- TR 持续放大 = 风力在增强(`t_us_tr_surge`);
- 缓跌阴线串 = 可能在退潮而非只是浪谷(温水煮青蛙,参见 [twenty_week_trend_system.md](twenty_week_trend_system.md) 的机械慢规则);
- 大浪是**叠加**出来的:疯狗浪(rogue wave)不是单一原因,而是多列波恰好同相——贵气×兑现×共振三层链找的正是"几列波即将同相"的位置;收益集中在少数爆发日的实证([return concentration](../t_us_return_concentration.py)),对应的就是海浪能量分布:大部分时间是涟漪,能量的大头在少数大浪里。

这是"指标是 GPS"的海洋版:**读浪是为了定位自己在风潮结构里的位置,不是为了预测下一朵浪花。**

### 2.4 水深 = 流动性

同样的能量进入浅水区,浪高剧增然后破碎。小盘、低成交的票就是浅滩——同一笔卖单能砸出深坑。`t_us_low_bounce` 的三道流动性门存在的理由:**不在浅滩上冲浪。**

### 2.5 无风有涌,涌浪更可信

涌浪(swell)可以从几千公里外的风暴传来——本地无新闻但价格持续单向推,能量来自别的板块或宏观资金面。**这往往比有新闻的行情更可信**,因为涌浪意味着源头风暴真实且巨大。板块轮动的"领跑→扩散"路径就是涌浪的传播。

## 3. 组合态:真正值钱的部分

单看任何一层都不如看三层的组合。四个典型海况:

| 海况 | 定义 | 市场对应 | 本系统实证/规则 |
|---|---|---|---|
| **風潮同向** | 牛市潮 + 顺势叙事风 | 涌浪整齐、周期长,冲浪黄金海况 | huice 里 NDX 池的正期望基本全部来自这里(edge 在池子,更在海况) |
| **風顶潮** | 风向与潮向相反 | 浪短、陡、碎,方向感全无 | **MIXED 体制 = huice 验证的唯一稳健负期望区**。不是信号变差了,是海况不适合任何船 |
| **无风有涌** | 本地无叙事但单向推 | 能量来自远方(跨板块/宏观) | 比有新闻更可信;sector_rotation 转强象限的本质 |
| **大风退潮** | 熊市里的暴涨叙事 | 每个浪都把你放在更低水位 | 熊反免疫期"仅提示不追"规则(regime_monitor × tech_swing 联动) |

## 4. 比喻失效的地方(同样重要)

### 4.1 海水总量其实会变,只是慢
长期看市场的"水"确实在增加:盈利增长、回购注销、央行扩表都是真实的水量变化,QE 就是往海里注水。**短期波动是能量现象(水量不变),长期趋势是水量现象。** 混淆两者会犯两种对称的错误:以为每天涨跌是"钱进钱出"(错把浪当水),或以为长期牛市也只是情绪(错把水当浪)。

### 4.2 浪不会引来風,市场会(反身性)
海浪再大不会改变天气;但价格上涨本身制造买入理由(动量、逼空、被动资金),下跌本身制造卖出压力(强平、止损级联)。**市场里浪能召唤风。** 这是纯物理比喻永远缺失的一环,也是趋势能自我延续到远超"合理"位置的原因——不要用物理直觉给趋势设"该到头了"的上限(那是讨债单思维)。

### 4.3 海浪统计平稳,市场会换海
同一片海域的浪高分布长期稳定;市场的波动分布会整体切换体制——2022 不是"浪大了一点",是**换了一片海**(huice: 2022 全部信号源 meanR 转负,右尾消失)。同一套浪形识别器在换海后全部失灵,所以需要 regime 层持续回答"这还是原来那片海吗"。

### 4.4 个股可以"呼出去就吸不回来"
海浪均值回归,幅度大致对称;个股可以退市、-89% 阴跌(CSCO 2000-02)。"跌多了必反弹"是把市场当成了纯波动系统。`t_us_low_bounce` 只把新低反弹当候选发现层而非验证信号,正是防这个陷阱。

## 5. 与本项目既有结论的呼应

这套心法不新增规则,但把散落的实证串成一幅图:

- **huice「edge 全在池子」** → edge 更全在**海况**:同样的浪形信号,風潮同向时正期望,風顶潮(MIXED)时负期望。池子(NDX)本身就是"季风常年吹的海域"。
- **MIXED 负期望** → 風顶潮碎浪区,唯一稳健的"不下水"信号。
- **温水煮青蛙 / 20周线** → 潮位仪:缓跌是退潮不是浪谷,机械慢规则的价值全在这里。
- **TR-surge 高位 TR_DOWN 最毒(α-13~-20)** → 风停浪未平 = 顶部派发;能量还在耗散但源头已断。
- **贵气×兑现×共振** → 多列波同相叠加的显式建模。
- **隔夜段独占方向信息** → 能量注入发生在闭市时(风在夜里吹),日内是浪的原地画圈。
- **收益集中在少数爆发日** → 海浪能量分布;据此钉死的退出参数(trim 走弱才减/gap 分档)= 别在大浪来之前离开海面。

## 6. 落地仪器

### 6.1 海况标签 (2026-07-14, registry #21)

框架的第一件仪器,把"海况决定期望"从事后解释变成**可证伪的前向记录**。`sea_state.py`:

- **潮向** = regime 快照 (`us_regime_state.json`,≤5 天): BULL/CAUTION=潮涨,WATCH/DEFEND=潮退。诚实声明:在真正的水量仪(tide gauge,落地方向二)建成前,这仍是价格代理。
- **风向** = 最新 sector_rotation 表 (≤7 天): 进攻组 (SMH/XLK/IGV/XLC/XLY/XBI) 与防守组 (XLP/XLU/XLV/XLRE) 的 rel21 中位差,>0=RISK_ON。全用已有列,不新造指标;不设死区——假说阶段少一个自由参数,贴零的边缘读数由 banner 印出分值自行判断。
- **象限**: ALIGNED 風潮同向 / CHOP 風頂潮 / EBB_RALLY 大風退潮 / EBB 風潮同退;任一输入缺失或过期 → UNKNOWN,不猜。
- **落点**: 晨报 MARKET STATE 块两行 banner (live only,--asof 不读——live 快照对历史日期是未来函数);`signal_ledger` 在 episode 开仓日集中盖章冻结 (所有写入方自动覆盖,旧行留空不回填);`t_us_signal_attrib` 周频按 signal_type × sea_state 分组。
- **判决条件**: 任一象限 n≥10 后首次按海况读 EV;若 ALIGNED 不优于 CHOP,回改的是本框架 (#20),不是仪器。

### 6.2 潮位仪·真水量 (2026-07-14, registry #22)

补上 §2.1 自我批评的仪器空缺——regime 的潮向是价格代理,系统里此前没有任何水量读数。`t_us_tide_gauge.py`(周日 cron),三个断面:

- **水源 (钱端,真水量)**: 净流动性 = Fed 总资产 WALCL − 财政部 TGA − 私方 ON RRP。数据源为 Fed H.4.1 官方 DDP zip (免 key;fredgraph 对本机不可达,FRED API 需 key;H.4.1 本就是这三个序列的原始出处)。潮向 = Δ13w 符号,无死区但幅度并印——历史刻度: 真抽水是 -3%~-11% (2022-03 -3.6%,2022-06 -11.3%),贴零读数是噪声。**首跑即给出背离样本 #1**: 2026-07-14 水量 EBBING(-0.2%,贴零) vs 价格潮位 RISING。
- **水位 (资产端)**: 全市场总市值,Nasdaq screener 全表个股 marketCap 求和 (首跑 $106.4T,Technology $44.8T 居首)。认识论戳:市值是**水位**(潮×浪合成读数)不是水量;Δ市值 ≈ 重定价(能量) + 净发行(真水),快照积累后减指数收益的残差 ≈ 净发行。
- **水流 (流向)**: "水从哪里流到哪里"——ETF 份额变动×价格 = 一级市场创赎美元额 (真金白银,非价格涨跌)。Yahoo 无 ETF 份额历史 → 前向积累快照 (29 只容器: 宽基股/板块/债/现金类/金/海外),第二次运行起出流向表。**股↔债↔现金间的搬家才是水量转移;板块间"轮动"多半是能量传递 (那是 sector_rotation 的岗位)。**
- **判决条件**: 背离样本 ≥3 次后复盘背离后 13/26w 的 QQQ 表现;水量仪证明领先性之前,sea_state 潮向**明确不换轨**,仍用 regime 快照。

### 6.3 阵风/季风分类器 (2026-07-14, registry #23)

给风层装记忆——§2.2 说阵风只值浪级响应、季风才值仓位级,但此前这个区分全靠人脑:rotation 是当日快照,Grok 每次都是新问句。`t_us_wind_class.py`(周日 cron),两个断面:

- **价格端 (客观,全史立即可算)**: 20 只 rotation ETF 对 SPY 的相对线,持续性 = 近 126 日贴近其 63 日 RS 高 (≤3%) 的天数占比,新鲜度 = RS 新高 ≤10 日或 rel21 ≥ +3%。二维合成五档:**季风**(新鲜×持续≥50%,仓位级) / **成势**(中间档,观察升级) / **阵风**(新鲜×持续<25%,只做单浪) / **季风间歇**(不新鲜×高持续,持有复查非新入场) / **无风**。`--asof` 校准:2024-06 SMH 读成势 49%(四五月回调后的再加速点,可辩护),XLI/XLF 季风间歇 92%/90% 准确捕捉 2024H1 宽度行情;bars 3 年窗 → 重放最早约 2 年 9 个月前。
- **叙事端 (Grok,前向积累)**: 周频 `narrative_scan`(~$0.1)列本周主导叙事(≤6 条);slug 是叙事的跨周身份——历史清单喂回 prompt 强制复用,匹配问题在生成端解决。周数计档:1=新阵风 / 2=成势 / ≥3=季风叙事;历史 ≥3 周的 slug 本周缺席 = **季风消退**(退风事件,相关持仓复查)。日志无法回填,越早积累越值钱(首周已录得 ai-capex 等 3 条)。
- **两端合读**: 两端印证 = 真季风;叙事有风价格无风 = 还没落地的故事;价格有风叙事无名 = **无风有涌,反而更可信**(落地方向四的入口)。
- **判决条件**: 叙事日志 ≥8 周后复盘"季风叙事期 vs 阵风期"的池内信号 EV;价格端阈值 (0.50/0.25) 是首版假说,**明确不为单例校准回调**。

### 6.4 无风有涌 SWELL 标注 (2026-07-14, registry #24)

框架推演出的、系统里原本没有对应物的信号形态——§2.5: 涌浪能量来自远方,比有新闻的行情更可信。`t_us_swell.py`(周日 cron),`t_us_low_bounce --grok` 的镜像(跌了查有催化剂 ↔ 涨了查无催化剂):

- **价格漏斗 (免费,全池)**: 涌的形状三道门——rel21 ≥ +8% (跑赢足够多) × 最大单日涨幅占累计 ≤40% (**碾磨门**: 单日暴动贡献大头的多半有名有姓,那是 gap_scan/earnings_react 的地盘) × 贴 63 日高 ≥95% (排除死猫跳,那是 low_bounce 的地盘)。池 = both (涌浪要在有主语的池子里找)。
- **名字复核 (Grok 前 8 名,~$0.8)**: `grok_lib.swell_scan` 问一个问题——这段涨幅在公开信息里有没有"名字"。news_intensity none/light = **SWELL**(无人问津的涌浪);heavy = 有名之风(正常行情)。
- **首跑发现**: 8/8 全 SWELL——碾磨门本身已经排掉了大多数有名之风,Grok 层目前是确认而非区分。这本身是信息(判决时对照组可能稀少,备选方案: 补"同涨幅非碾磨"对照组)。
- **判决条件**: swell_log 成熟样本 ≥10 后首次统计 SWELL vs 有名之风的 fwd 表现。n=0 假说,纯标注,不构成入场信号(入场仍走各扫描器的 Setup 纪律)。

## 7. 一页速查

| 问题 | 海洋答案 |
|---|---|
| 钱去哪了/从哪来? | 短期没有钱的增减,只有能量传递。别数水,看风和潮 |
| 这个指标在测哪一层? | 先贴标签:潮位仪 / 风向标 / 浪形仪,别混用 |
| 日线形态判牛熊? | 拿浪高测潮位,尺度错配。潮要看水量指标 |
| 这条新闻值得响应吗? | 先分阵风还是季风:阵风浪级响应,季风才动仓位 |
| 无新闻的单边推? | 涌浪,比有新闻更可信,找远方的风暴 |
| MIXED 体制要不要进? | 風顶潮碎浪区,实证负期望,不下水 |
| 熊市里的暴涨叙事? | 大风退潮,浪再高也在更低水位,仅提示不追 |
| 趋势"该到头了"? | 浪能召唤风(反身性),别用物理直觉设上限 |
| 小票同样的信号? | 先量水深(流动性),浅滩浪形失真且易碎 |

> **交易浪、分析風、服从潮——并且永远先问一句:现在是什么海况?**
