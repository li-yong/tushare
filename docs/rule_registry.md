# Rule Registry — live 规则的有效域与闭环状态

来源:《实践论》《矛盾论》复盘 (2026-07-11)。解决两个认识论问题:

1. **第二次飞跃闭环**(理性认识必须回到实践):每条研究/回测/归因结论,必须以
   两种结局之一收尾 —— **已改**(改了哪个门控/参数)或 **明确不改**(+理由)。
   没有这一行的结论视为未完成的认识,不许只躺在 findings 文档里。
2. **有效域戳**(真理是具体的、历史的):每条 live 规则标注它在哪个池子、哪种
   体制、哪段数据上验证过。参数是带保质期的认识,不是永恒真理;体制切换或
   到达复验触发条件时进复验队列,而不是等亏钱才想起来查。

**认识等级制**(不许倒置):huice 合成重放 = **假说**;live ledger
(`t_us_signal_attrib.py` 周频归因)= **知识**。huice 里再漂亮的 α,ledger
前向样本不够就只配"候选"待遇;ledger 持续打脸的信号,huice 说什么都不算数。

维护方式:改任何 live 门控/参数、或复盘得出新结论时,更新本表相应行(或加行)。
周日复盘顺带扫一眼"复验触发"列。

## 错误归类速查 (复盘用)

犯错先归类是哪一"倾",两种病药方相反,混着治会来回摆:

- **右倾 = 认识落后于实际**:体制已切换还拿旧规则做(2022 型)。药方:缩短复验
  周期、检查主要矛盾是否过期。
- **左倾 = 超过客观过程的阶段**:转化条件未出现就抢跑预期中的反转(讨债单思维)。
  药方:给论断绑可观测条件(surge_stopline 型转化条件探测器),条件没命中不动手。
